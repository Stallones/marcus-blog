# 问题追踪：本地 Markdown

本仓库的问题和 PRD 以 Markdown 文件形式存放在 `.scratch/` 目录下。

## 约定规范

- 每个功能一个目录：`.scratch/<功能名称>/`
- PRD 文件：`.scratch/<功能名称>/PRD.md`
- 实现问题：`.scratch/<功能名称>/issues/<编号>-<名称>.md`，编号从 `01` 开始
- 分类状态记录在每个问题文件顶部的 `状态:` 行中（角色字符串见 `triage-labels.md`）
- 评论和对话历史追加到文件底部 `## 评论` 标题下

## 技能要求"发布到问题追踪器"时

在 `.scratch/<功能名称>/` 下创建新文件（目录不存在则自动创建）。

## 技能要求"获取相关工单"时

读取指定路径下的文件。通常由用户直接传入路径或问题编号。
