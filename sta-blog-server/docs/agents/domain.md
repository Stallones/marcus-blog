# 领域文档

工程技能在探索代码库时如何消费本仓库的领域文档。

## 探索前先阅读

- 仓库根目录下的 **`CONTEXT.md`**
- 如果存在 **`CONTEXT-MAP.md`**，则指向每个上下文的 `CONTEXT.md`。阅读与你即将工作领域相关的每个文件。
- **`docs/adr/`** — 阅读与你即将工作领域相关的架构决策记录。

如果以上文件不存在，**静默继续**。不要标记它们的缺失，也不要建议提前创建。

## 文件结构

单上下文仓库：

```
/
├── CONTEXT.md
├── docs/adr/
│   ├── 0001-event-sourced-orders.md
│   └── 0002-postgres-for-write-model.md
└── src/
```

## 使用词汇表中的术语

当你的输出命名一个领域概念时（如在问题标题、重构建议、假设中），使用 `CONTEXT.md` 中定义的术语。不要使用词汇表明确避免的同义词。

## 标记 ADR 冲突

如果你的输出与现有 ADR 相矛盾，请明确地将其提出，而不是静默覆盖。
